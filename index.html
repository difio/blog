FAQ

Posted by Monupco Team @
Jan 30th, 2012

Categories: News

What is this

Monupco is a service which let’s you know what package updates are available for your applications deployed in the cloud. We try to provide meaningful and accurate information whenever possible by monitoring various sources of information.

Why should I care

In a cloud environment you care about your own code and leave updates to the vendor. However applications install 3rd party dependencies which are not part of the vendor platform. These packages may present security vulnerabilities or incompatibilities and disruption of service if left unmaintained. This is why Monupco exists.

How much does it cost

The service will remain free of charge during Alpha and Beta versions. We may need to charge a small ammount afterwards. Don’t worry it will not be more than a few bucks.

How to install

The client side code for Monupco is open source and is split into different packages depending on cloud vendor and framework type. For detailed install instructions see the README files in our GitHub repositories at https://github.com/monupco.

Why do I need to approve applications

Applications are registered on behalf of your username only because we don’t want you to accidentally commit the password in your source code. To ensure that you registered the application it has to be approved first. This is similar to the account approval process on some websites.

When is application data updated

Data for packages installed on your applications is tipically updated when you push a new version of the application which may pull in new dependencies for example.

Why some updates do not show immediately

New versions are added periodically by an automated process. Data is then reviewed and scrutinized by a human to ensure it is correct.
